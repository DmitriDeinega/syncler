### 2. Phase 3c MuteStore Semantics
**Status: ACCEPTABLE (V1 tradeoff)**
The reviewers agree that while `StateMerger` remains monotone (union-only mutes), the **local-override layer** in `MuteStore` effectively mitigates the issue for the acting user. Since "unmute" forces a local override to `false`, the sender will remain visible on the device where the user took action, even if synced state resurrects the "muted" flag from another device. This is a solid V1 compromise that avoids the complexity of tombstones for simple preference flags.

### 3. Commit Strategy: Unified Phase 3 Landing
**Recommendation: UNIFIED**
**Rationale:** The sub-phases are now tightly coupled: the inbox UI (`3c`) depends on the V5 schema for mute filtering, which is used across both template (`3a`) and live-card (`3b`) views. The security hardening in `TemplateActionRunner` and `upsert_live_card` makes the combined feature set safe for master. A split landing would require temporary shims for the shared `nonMuted` filtering logic.

### 4. Tests Required Before Commit
The following tests must be present in the final commit to prevent regressions:

**Server (`pytest`)**
*   **`test_upsert_revoked_plugin_rejected`**: Verify 410 Gone when `Plugin.revoked_at` is set.
*   **`test_upsert_missing_pairing_rejected`**: Verify 401/404 when no active `Pairing` exists for `(sender_id, user_id)`.
*   **`test_upsert_ttl_clamping`**: Verify `expires_at` > 48h is clamped or rejected (per `ExpiredEnvelopeError`).
*   **`test_delete_cross_user_rejected`**: Attempt to delete a card using a signature valid for User A against User B's card.
*   **`test_upsert_sequence_regression`**: Verify 409 Conflict when `sequence_number` is not strictly increasing.

**Android (`JUnit/Compose`)**
*   **`TemplateActionRunnerTest`**: Verify that `post()` does NOT include the `Authorization` header and rejects non-HTTPS endpoints in release.
*   **`MuteStoreTest`**: Verify the effective set logic: `(synced + local_mute) - local_unmute`.
*   **`InboxRepositoryRevokeTest`**: Verify `revokeSender` stops and fails on network error vs. proceeding to local delete only on success.
*   **`InboxScreenMuteTest`**: UI test verifying a message from a muted sender is hidden in both "All" and "Archive" views.

### 5. New Concerns / Regressions

*   **SDK Canonicalization**: Ensure the Python SDK's `delete_card` signature matches the new `(sender_id, user_id, card_key)` envelope exactly.
