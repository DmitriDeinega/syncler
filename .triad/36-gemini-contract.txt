This contract defines the structured metadata embedded in the encrypted payload under the reserved `hostPreview` key. This allows the host application to render inbox rows natively without waking up the plugin.

### 1. TypeScript interface (`sdk-plugin`)

```ts
/**
 * Structured metadata used by the host to render inbox rows natively.
 * Embedded in the encrypted message payload under the `hostPreview` key.
 */
export interface HostPreview {
  /** Main bold text for the inbox row. Required. Max 64 chars. */
  title: string;
  /** Secondary text shown below the title. Max 128 chars. */
  subtitle?: string;
  /** Brief snippet for expanded views or notifications. Max 256 chars. */
  summary?: string;
  /** Hints for sorting or highlighting. Defaults to 'normal'. */
  priority?: 'low' | 'normal' | 'high' | 'urgent';
  /** 
   * Keywords for the host's global search index. 
   * Max 10 tokens, 32 chars each.
   */
  searchText?: string[];
  /** 
   * Optional ISO-8601 timestamp to override the display time. 
   * Useful for historical data or event-specific alerts.
   */
  timestamp?: string;
  /** 
   * Custom screen-reader label. 
   * Falls back to "[title], [subtitle]" if absent. 
   */
  accessibilityLabel?: string;
}
```

### 2. Python `TypedDict` (`sdk-python`)

```python
from typing import TypedDict, List, Literal, Optional

class HostPreview(TypedDict, total=False):
    """
    Structured metadata for native host rendering.
    
    The sender passes this inside the payload dict to client.send_to().
    Example: client.send_to(..., payload={"hostPreview": {...}, "data": ...})
    """
    title: str               # Required (semantic), max 64 chars
    subtitle: str            # Optional, max 128 chars
    summary: str             # Optional, max 256 chars
    priority: Literal["low", "normal", "high", "urgent"]
    searchText: List[str]    # Max 10 tokens, 32 chars each
    timestamp: str           # ISO-8601 format string
    accessibilityLabel: str  # Screen reader override
```

### Constraints & Justification

*   **Required vs optional**: `title` is the only mandatory field. A row must have at least one identifying string.
*   **Field length caps**: 64/128/256 chars. These limits ensure the metadata fits mobile layouts and keeps the decryption/parsing overhead predictable.
*   **`searchText`**: `string[]` (tokens). Providing tokens directly allows the host to build a simple prefix-index or set for global search without complex parsing.
*   **Priority**: Enum (`low`, `normal`, `high`, `urgent`). Allows the host to pin urgent alerts or mute low-priority noise.
*   **Category**: **Omitted**. Plugin-level filtering is sufficient; per-message categorization is redundant for a 2-line inbox row.
*   **Icon / color**: **Omitted**. Brand identity is derived from the sender's verified name and avatar.
*   **Timestamps**: `timestamp` (optional ISO string). Enables accurate "at a glance" timing for historical imports or delayed alerts.
*   **Accessibility**: `accessibilityLabel` provides a flat string for screen readers, preventing "jumbled" reading of nested metadata.
*   **Fallback**: If `hostPreview` is missing (legacy sender), the host renders: "New message from [Sender Name]".
*   **Payload size cap**: **2KB**. The total `hostPreview` object must not exceed 2048 bytes to ensure fast inbox processing during message polling.
