=== BATCH 2 QUESTIONS ===
1.  For the trading bot reports, must the server be blind to their contents (E2E encrypted), or can it see the data in transit? A "yes, it can see the data" answer simplifies V1, but a "no, it must be blind" answer makes the platform fundamentally more secure.
2.  Your trading bot might generate a Plotly chart. For V1, should we render this with a simple, sandboxed HTML view, or must we build the entire third-party plugin system now just for that one chart type?
3.  The user taps "played" on the lottery card while their phone is offline. When they reconnect an hour later, does the lottery app's SDK need to handle retries and de-duplication, or is that the platform's job?
4.  Your trading bot restarts and sends 50 duplicate reports. Should the user's phone get 50 notifications, or should the platform recognize them as duplicates and show only one?
5.  How does the trading bot on AWS prove its identity to send a card? Does it use a simple API key that can send anything, or a signed JSON Web Token (JWT) for each card that proves it came from the bot and hasn't been tampered with?
6.  For the lottery app's Python SDK, which is the better V1 implementation: a generic `sdk.send(card_object)` or a purpose-built `sdk.request_confirmation(prompt, data)`? The latter is an easier integration for the sender, but it commits the platform's API to a specific "confirmation" concept.
7.  For the spec-doc handoff test, what's the maximum number of pages that document can be? If a developer needs more than that to integrate a simple "played" button, we have failed the "good developer experience" test.
8.  When the user taps "played," how does the lottery app receive that response? Does it need to host a public webhook endpoint for our server to call, or will it maintain a persistent connection to our server to receive the event directly?
9.  How does the app's UI distinguish between a card from the trading bot and a card from the lottery app? Does each sender get a unique name and icon in the feed, and is setting that up part of the V1 workflow?
10. Does the platform need to support scheduled sending? For instance, should the trading bot be able to tell the platform "send this report every hour," or is the bot solely responsible for its own timing via its own cron job?
