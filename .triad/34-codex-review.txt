**Findings**

- **P0: Inbox render path can execute unsigned bundle JS, including over HTTP in release.**  
  [InboxRepository.kt](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/InboxRepository.kt:87) fetches `latest.signedBundleUrl` and executes the returned JS without checking `bundle_hash` or `signature`, and without a debug-gated HTTPS check. [PluginPublishRequest](D:/Projects/syncler/server/app/schemas.py:301) also accepts any `signed_bundle_url`. Combined with app-wide `usesCleartextTraffic="true"` in [AndroidManifest.xml](D:/Projects/syncler/android/app/src/main/AndroidManifest.xml:12), a release build can fetch and execute a substituted HTTP bundle in this path. For dogfood debug, document this explicitly; before prod, reuse `PluginSignatureVerifier`/hash check and require HTTPS unless `BuildConfig.DEBUG`.

- **P0: `NetworkBridge` release HTTP relaxation is bypassed in the existing plugin host.**  
  [NetworkBridge.kt](D:/Projects/syncler/android/feature/plugin-host/src/main/kotlin/app/syncler/android/pluginhost/capabilities/NetworkBridge.kt:21) only applies `MODERN_TLS` in release when it constructs its own client. But [PluginLoader.kt](D:/Projects/syncler/android/feature/plugin-host/src/main/kotlin/app/syncler/android/pluginhost/PluginLoader.kt:107) injects a default OkHttp client, so release `platform.network.fetch` can still use OkHttp’s default cleartext-capable specs if the manifest declares an `http://` endpoint. Fix by enforcing scheme in `fetch()` itself, or always wrapping injected clients with release-safe `connectionSpecs`.

- **P1: `CardBridge` lifecycle leak is real.**  
  [InboxScreen.kt](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/InboxScreen.kt:143) creates a `WebView` and local `CardBridge`, but there is no `AndroidView` `onRelease`/disposal path. [CardBridge.kt](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/CardBridge.kt:36) owns a coroutine scope, and [destroy()](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/CardBridge.kt:67) is never called. Add disposal that removes the JS interface, cancels the bridge, stops loading, and destroys the WebView. The `@JavascriptInterface` annotation itself is present at [CardBridge.kt:48](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/CardBridge.kt:48).

- **P1: AAD reconstruction is byte-correct for the Python SDK, but timestamp canonicalization is fragile.**  
  Android uses `dto.expiresAt` directly in AAD at [InboxRepository.kt](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/InboxRepository.kt:110). The server verifies using `payload.expires_at.isoformat().replace("+00:00", "Z")` at [messages.py](D:/Projects/syncler/server/app/routers/messages.py:65), then later returns the DB datetime at [messages.py](D:/Projects/syncler/server/app/routers/messages.py:246). If any sender uses a non-UTC offset string that the server accepts, Postgres may normalize it before inbox response and device decrypt will fail. For tonight’s Python SDK dogfood, `Z` output is okay. For protocol safety, reject non-UTC `expires_at` or persist/return the exact canonical AAD string.

- **P1: Docs still contradict V1 action callback behavior.**  
  [integration-guide.md](D:/Projects/syncler/docs/integration-guide.md:42) says `device_id` is in the callback body and [line 46](D:/Projects/syncler/docs/integration-guide.md:46) says dedupe by `(item_id, device_id)`, but [line 218](D:/Projects/syncler/docs/integration-guide.md:218) correctly says V1 does not inject `device_id`. The Flask example still reads `data["device_id"]` at [line 229](D:/Projects/syncler/docs/integration-guide.md:229). This will break a sender following the guide.

- **P2: One SDK example script still builds ESM.**  
  `sdk-plugin/examples/minimal/build.sh` is fixed, but `npm run build:example` in [package.json](D:/Projects/syncler/sdk-plugin/package.json:15) still uses `--format=esm`. Anyone using that script gets the old non-module `<script>` syntax error.

**Validated**

- `plugin_publish` rate limiting is shaped correctly: the per-sender bucket fires only after sender lookup and signature verification, so spoofers cannot burn another sender’s bucket.
- FCM/storage separation looks clean: storage gates on active devices, push remains best-effort for FCM-equipped devices.
- `plugin_identifier` projection is batch-loaded for inbox and falls back safely.
- `min_plugin_version` null-to-empty handling matches the Python SDK AAD shape.
- `npm run build` in `sdk-plugin` passes.

Server regression tests did not run locally because the configured Postgres credentials failed (`InvalidPasswordError` for user `syncler`).
