**Findings**

[P2] [EncryptedUserState.kt:55-57](/absent/path/not-used) keeps `deleted_messages` under `SCHEMA_V2`. I would hold for a `SCHEMA_V3` bump if any M11.5/V2 APK can still sync. An older V2 client will parse the blob, ignore `deleted_messages`, and later write back schema 2 without that field, resurrecting deleted messages on M11.6 devices. The existing future-schema push guard only protects you if this field moves to V3.

[P2] `TimestampFormat.relative()` accepts `now`, but [TimestampFormat.kt:34](D:/Projects/syncler/android/feature/inbox/src/main/kotlin/app/syncler/feature/inbox/TimestampFormat.kt:34) uses `LocalDate.now(zone)` instead of deriving today from the supplied `now`. Any deterministic tests with a fake `now` become date-dependent/flaky, and date-branch behavior can disagree with the delta calculation. Use `val today = now.atZone(zone).toLocalDate()`.
