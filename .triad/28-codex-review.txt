=== AGREEMENTS WITH FIXES ===
S1: concern: AAD/envelope split is structurally correct in `aead.py` and Android vectors, but `expires_at` is now signed/AAD material and is not returned by inbox/detail responses. Recipients cannot reconstruct AAD/envelope for decrypt/verify. See `server/app/schemas.py:167` and `server/app/routers/messages.py:201`.

S2: ok on ordering: signature verify → nonce check → store. Concern: `registry.seen()` marks the nonce before `store_message()`, so a valid signed request that later fails validation/DB commit can burn the nonce.

S3: ok. `_plugin()` now enforces `Plugin.sender_id == sender_id`.

S4: ok. Hilt stubs exist for the missing push bindings.

S5: ok for the intended no-FCM-token path. The test creates an active device without `fcm_token`, then sends a valid signed message and gets 410. Minor concern: no-device and no-FCM-device are not actually distinguished in service behavior.

F1: concern. `expires_at` is required and signed, and past/>30d is checked, but timezone-naive datetimes can still reach `store_message()` and cause a naive/aware datetime comparison 500 instead of a 4xx. Also `expires_at` is missing from recipient message responses.

F2: concern. `/send` has the IP bucket, but the previous sender bucket is no longer wired despite the comment saying it applies after signature verification. Also `_client_ip()` trusts `x-forwarded-for` unconditionally, so direct clients can spoof around the IP limit unless a trusted proxy strips/sets it.

F3: ok. Dismiss now checks device ownership.

F4: ok implementation-wise. Inbox/get filter `Message.expires_at > now`.

F5: mostly ok. Active job tracking and supervisor cancellation are present, timeout posts delivery-failed. Concern: exceptions thrown by `pipeline.process()` skip the stop/failure path.

F6: ok. `SynclerFcmService` cancels its supervisor on destroy.

=== NEW BUGS INTRODUCED ===
1. Show-stopper: `expires_at` is part of AAD/envelope but is absent from `MessageInboxItem` / `MessageDetailResponse`, so clients cannot decrypt or verify fetched messages.

2. `/send` lost the per-sender `message_send` rate limit; only `message_send_ip` is applied.

3. `expires_at` accepts timezone-naive datetimes, which can produce server 500s during retention comparisons.

4. Nonce replay registry burns nonce before durable store succeeds.

5. `PluginNotificationService` does not handle pipeline exceptions; failed jobs may not post failure or stop the service cleanly.

=== STILL OPEN FROM M5 REVIEW ===
Most original items are addressed, but rate limiting is still incomplete because the authenticated sender/user buckets are not wired after signature verification.

=== SHOW-STOPPERS FOR M6 ===
Add `expires_at` to inbox/detail responses and tests, because encrypted recipients need it to reconstruct AAD/envelope.

I would also fix timezone validation before M6 if this endpoint is exposed to non-SDK callers.
