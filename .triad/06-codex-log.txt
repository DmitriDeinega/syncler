### Lead's architectural read ??? Bitwarden / ProtonMail zero-knowledge model

- User signs up with email + password. UUID assigned.
- Password derives an encryption key client-side (Argon2id). Server never sees password or key.
- Server stores everything encrypted under user UUID: plugin install state, sender pairings, message ciphertext.
- New device: email + password ??? device pulls encrypted user blob ??? decrypts locally ??? ready.

- Lost devices AND forgot password = data gone. Recovery seed phrase is V1.5.
- Server is blind to message *content* but knows *metadata* (account exists, plugin installed, sender paired, message 
flowed at T). Explicit, not hidden.
