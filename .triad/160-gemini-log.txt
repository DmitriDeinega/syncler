### Item 4 — Android JWT Test

*   **Response:** **Extract the factory to a named class.** Testing anonymous objects through `PluginLoader.android()` is brittle and creates high-ceremony setup for a simple logic check. ### Item 5 — Affected-Senders Inference
*   **Verdict:** **DESIGN**.
*   **Response:** **Support the simplification.** Listing all `paired_senders` from the `PairedSenderStore` is the most robust v0.1 approach. Precise inference via V2 envelope inspection is complex and potentially "leaky" if historical data is missing. A "Security Audit" UX—where the user reviews all pairings following a rotation—is safer and more transparent.

### Integration Gap Observation
A partner project would benefit from a simple `client.whoami()` or `client.ping()` helper in the Python SDK. While Item 3 provides server discovery, there is no typed way to verify "Is my API Key / Sender ID valid?" without attempting a mutation. Adding a `client.get_sender_info()` (hitting `/v1/senders/{id}`) would close the "first-run validation" loop for developers.
