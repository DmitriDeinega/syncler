### 3. Review of Concerns

1.  **HPKE aad always-empty:** **Acceptable.** RFC 9180 ensures that the `info` string is mixed into the KDF (Key Derivation Function). Binding context via `info` is functionally equivalent to `aad` for domain separation and authentication context in this design. Since PyCA 48+ facilitates this via a single-shot API, sticking to `info` for all metadata is the correct path for interop.
2.  **Verify-before-trust ordering:** **Adequate.** The `sender_id` is treated as a lookup key for a trusted public key. If a sender ID is spoofed, the signature verification will fail. While CPU-intensive (Ed25519) before rate-limiting, it is secure for V0.1.
3.  **Recipient classifier matrix:** **Exhaustive.** Row 5 (409 on stale version) is the critical safety valve. Tolerating "recently_revoked" extras (Row 6) is a smart way to handle the propagation delay of revocation without breaking the user experience.
4.  **Migration safety:** **Acceptable.** For V0.1/Alpha, "wipe-on-upgrade" for non-persistent caches (live cards/messages) is standard practice.
5.  **Pointer storage:** **Acceptable.** Using JSON in a `TEXT` column provides high visibility and easy debugging during Phase 9. If bandwidth becomes a bottleneck, moving to a binary `BLOB` or a dedicated table can be an optimization for V1.0.
6.  **SDK retry pattern:** **Aggressive enough.** A single retry on 409 covers the most common race condition (device rotation during message prep). Adding backoff/multi-retry logic is only necessary if the system encounters high-frequency concurrent rotations for single users.
7.  **DeviceEncryptionKeyStore:** **Optimal.** Since Android Keystore's TEE support for X25519 is inconsistent across manufacturers/API levels, software-based BC keys inside `EncryptedSharedPreferences` (which uses Keystore for its wrapping key) is the idiomatic solution.
8.  **Android HPKE library choice:** **GREEN.** BouncyCastle 1.80 is a heavy-weight but standard-compliant choice. Since it's already in the classpath, there is no reason to pull in Tink's additional footprint.
