### Item 1 — Settings UI revoke
**Verdict: DESIGN**

*   **Pivot:** The per-plugin pivot is correct for v0.1. It aligns with how users think about "what is this plugin doing?"
*   **OS-permission interaction (Concern 1):** Do **not** attempt to automatically revoke OS-level permissions. Android's permission model doesn't allow apps to revoke their own permissions programmatically (beyond `revokeSelfPermissionOnKill` which is volatile). Forcing the user into `ACTION_APP_DETAILS_SETTINGS` automatically is jarring.
    *   **Recommendation:** If the user revokes the *last* plugin grant for an OS-backed capability (e.g., Camera), show a "Privacy Tip" dialog or snackbar: *"No more plugins are using the Camera. You can revoke Syncler's system-wide Camera permission in [System Settings]."*
*   **Last-used hint (Concern 2):** **OK.** Definitely show this. Since `PluginCapabilityGrantRow` already tracks `lastInvokedAtMs`, this adds high perceived value for negligible performance cost.

### Item 2 — Inbox UI plugin.onAction routing
**Verdict: FIX**

*   **Fallback semantic (Concern 3):** The proposed fallback (*"If not loaded → fire-and-forget"*) is too aggressive. It creates a "race against the garbage collector"—if the plugin was recently unloaded, the action bypasses the `onAction` hook, potentially breaking plugin-enforced logic or state.
*   **Recommendation:** Attempt to load the plugin.
    1.  If plugin is **installed but unloaded**: Attempt load (with a short timeout, e.g., 2s).
    2.  If plugin is **not installed**: Fall back to fire-and-forget POST (preserves V1 behavior for "orphaned" cards).
    3.  If load **fails/times out**: Only then fall back to fire-and-forget, with an `AuditLogger.record` entry.

### Item 3 — AuditLogger rename
**Verdict: OK**

*   **Deprecation (Concern 4):** Since this is an internal API transition (v0.1 → v0.2), skip the deprecation window. **Batch-rename all ~30 call sites** in a single clean commit. It keeps the codebase "debt-free" and the logs accurate immediately.

### Item 4 — Compose golden tests
**Verdict: OK**

*   **Approach (Concern 5):** **Option B (Semantics tests)** is the winner. These layouts are structural. Verifying that the correct `Text` and `Button` nodes exist with the expected content provides 90% of the safety for 10% of the maintenance cost compared to Paparazzi/Snapshots.
