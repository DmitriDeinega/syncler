Implemented M3 under `android/` only.

**Summary**
- Added Gradle multi-module Android project:
  `:app`, `:core:auth`, `:core:crypto`, `:core:network`, `:core:storage`, `:feature:inbox`, `:feature:settings`.
- Added Compose Material 3 app shell with signup/login, inbox stub, logout, and device management/revoke screen.
- Added Retrofit + Moshi API client for the FastAPI routes.
- Added Hilt DI across app/core modules.
- Added secure storage via `EncryptedSharedPreferences` and placeholder SQLCipher Room DB.
- Added crypto primitives and unit tests for Argon2id, AES-GCM, HKDF, Ed25519, auth, and network.

**Versions**
- Kotlin: `2.1.21`
- Argon2: `com.lambdapioneer.argon2kt:argon2kt:1.6.0`
- Target SDK: `35`
- Min SDK: `26`

**Spec Notes**
- Tightened one missing V1 detail: the current server has no pre-login salt endpoint, but login requires `auth_key_hash` before returning `auth_salt`. I made Android derive a deterministic 16-byte auth salt from normalized email for V1 so signup/login work against the existing server.
- Loosened Ed25519 device key storage for Android 8-11: Android Keystore Ed25519 is gated to API 31+, with an encrypted fallback seed in `EncryptedSharedPreferences` for minSdk 26 support.
- Master key is kept in memory only. JWT and local DB passphrase are encrypted at rest.
