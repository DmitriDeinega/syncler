```ts
// sdk-plugin
export interface HostPreview {
  /**
   * Required. Primary inbox row text.
   * Max 80 UTF-8 bytes after trimming.
   */
  title: string;

  /**
   * Optional. Secondary row text, usually sender-local context:
   * account, ticket, market, workflow, or short status.
   * Max 120 UTF-8 bytes.
   */
  subtitle?: string;

  /**
   * Optional. One compact human-readable sentence shown when the row has room.
   * Max 240 UTF-8 bytes.
   */
  summary?: string;

  /**
   * Optional. Extra normalized search terms for host-wide search.
   * Max 16 entries; each entry max 64 UTF-8 bytes.
   * The host indexes these in addition to title/subtitle/summary.
   */
  searchText?: string[];
}
```

```python
# sdk-python
from typing import TypedDict, NotRequired

class HostPreview(TypedDict):
    # Required. Primary inbox row text. Max 80 UTF-8 bytes after trimming.
    title: str

    # Optional. Secondary row text. Max 120 UTF-8 bytes.
    subtitle: NotRequired[str]

    # Optional. Compact preview sentence. Max 240 UTF-8 bytes.
    summary: NotRequired[str]

    # Optional. Extra host-search terms. Max 16 entries, 64 UTF-8 bytes each.
    searchText: NotRequired[list[str]]
```

**Yes/No Decisions**

- Required fields: **Yes**. Only `title` is required. Missing optional fields still render a sensible row.
- Field caps: **Yes**. `title` 80 bytes, `subtitle` 120, `summary` 240, `searchText` 16 x 64. Enough for dense inbox rows, bounded for polling.
- `searchText`: **Yes, array of tokens/phrases**. The sender knows domain terms best; the host indexes these plus visible text.
- Priority: **No**. Omitted. The host should not let senders silently influence row ordering or alarm styling.
- Category: **No**. Omitted. Too plugin-specific; use `subtitle` or `searchText`.
- Icon/color: **No**. Omitted. Visual identity comes from sender/plugin avatar and sender name.
- Timestamps: **No sender timestamp field**. Row displays server `sent_at`. If event time matters, put it in `subtitle` or `summary`.
- Accessibility label: **No**. Omitted. Host derives accessible text from sender name, sent time, title, subtitle, and summary.
- Missing `hostPreview`: **Yes fallback**. Host renders sender/plugin name as title, server `sent_at`, and generic text: `Encrypted message`.
- Payload size cap: **Yes**. Serialized `hostPreview` must be <= `1024` UTF-8 bytes. Larger previews are rejected or ignored.
