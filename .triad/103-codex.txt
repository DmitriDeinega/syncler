The remaining issue is a small but real wording bug in §10.8 step 7:

> “Compute the request’s deduped pairing-id set. If the two sets differ … or duplicate request ID…”

A deduped set cannot detect the case where the request contains a duplicate but no missing ID.

Example:

- canonical active set: `{A, B}`
- request IDs: `[A, A, B]`
- deduped request set: `{A, B}`

Set equality passes, but the spec says duplicate request ID must produce `409 pairing_set_changed`.

Fix: add an explicit duplicate check before set equality, e.g.:

> First verify `len(raw_request_pairing_ids) == len(deduped_request_pairing_ids)`; otherwise respond `409 pairing_set_changed`. Then compare the deduped request set to the canonical active set.
