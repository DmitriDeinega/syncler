### Answers to Questions A–H

**A. Handle string scoping enforcement**
**Answer:** The UUID + `(plugin_row_id, handle)` DB scoping is perfectly sufficient. A Type 4 UUID has 122 bits of entropy; brute-forcing it within a 5-minute TTL locally is computationally unfeasible. No HMAC or prefix required. 

**B. Handle TTL clock source**
**Answer:** FIX: Use `SystemClock.elapsedRealtime()` for enforcing the TTL expiration. Using wall-clock (`currentTimeMillis`) for TTLs is a known vulnerability to NTP shifts or manual timezone changes. (You can retain wall-clock for UI/audit logs, but the expiration check must use the monotonic clock).

**C. fileBytes chunk re-read semantics**
**Answer:** Stateless seek-and-read is exactly what you want. The host should not maintain open cursors or stream state for the plugin. Every IPC call provides `offset` and `length`, and the host opens, seeks, reads, and closes (or uses `RandomAccessFile`).

**D. Camera tempfile lifecycle interplay**
**Answer:** Confirm immediate deletion. Once the host successfully copies the `ACTION_IMAGE_CAPTURE` tempfile to the `cacheDir/plugin-capability/<token>/` staging file, the original camera tempfile should be wiped immediately. The plugin's handle lifecycle applies *only* to the staging copy.

**E. Audit row schema**
**Answer:** Your proposed schema is good. Final fields should be: `plugin_row_id`, `capability_name` (e.g., "gallery"), `outcome` (string/enum from your taxonomy), `at_ms` (wall-clock for display), and `sandbox_token`. No handles, coordinates, or URIs.

**F. NATIVE_SDK_ABI bump messaging**
**Answer:** Stick to the existing load-failure UI (`unsupported_sdk_abi`). Since there are no V0.1 native plugins in the wild, building a custom settings banner for this edge case is wasted effort right now.

**G. Server-side capability validation**
**Answer:** Reject manifests declaring both. Enforce single-granularity declaration. If they want fine with coarse-fallback, they should declare `location.fine` and rely on the Android OS precision downgrade + the new `precision` field in the SDK return type.

**H. Photo Picker single vs multi**
**Answer:** Branch based on `maxItems`. Use `PickVisualMedia` for `maxItems == 1` and `PickMultipleVisualMedia` for `> 1`. Hard-cap `maxItems` at 10 (or similar) at the SDK boundary to prevent the plugin from exhaustively staging too much data and hitting the 16MB/32-handle limits instantly.
