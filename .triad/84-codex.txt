Action endpoint check is not commit-ready. There are still SDK/server divergence cases in the actual implementation:

- [server/app/schemas.py](/abs/path/D:/Projects/syncler/server/app/schemas.py:475) uses `re.IGNORECASE` without `re.ASCII`. Python case-insensitive ranges match extra Unicode letters, while JS does not. Server accepts these HTTPS hosts; SDK rejects them:

  - `https://İ.example/x`
  - `https://ı.example/x`
  - `https://ſ.example/x`

- Python `\d` in the port also matches Unicode digits. Server accepts `https://api.example:١٢/x`; SDK rejects it.

- Python `re.IGNORECASE` also lets the scheme match Unicode-folded `s`. Server accepts `httpſ://localhost/x` because host is `localhost`; SDK rejects it.

**Over-restrictive cases**

The scheme says “HTTPS always,” but the regex rejects legitimate HTTPS URL hosts such as bracketed IPv6 literals: `https://[::1]/x` or `https://[2001:db8::1]/x`. If IPv6 is intentionally out of scope, document that. If not, this is another compatibility gap.
